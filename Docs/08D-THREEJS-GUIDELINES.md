# EcoLoop Three.js Guidelines

## Purpose

Three.js exists to create memorable moments.

Not entire pages.

---

## Allowed Areas

### Landing Hero

Particle atmosphere.

### Dashboard Background

Environmental field visualization.

### Action Completion

Micro celebration.

### Empty States

Ambient visualizations.

---

## Forbidden Areas

* forms
* settings
* tables
* profile pages
* onboarding

---

## Performance Budget

Desktop:

60 FPS

Mobile:

45 FPS

Fallback:

Static Canvas

---

## Particle System

Maximum:

5000 particles desktop

1000 particles mobile

---

## Accessibility

Every Three.js experience must have:

* reduced motion mode
* static fallback
* keyboard accessibility

No critical information may exist only inside a 3D scene.
