# EcoLoop — Visual References

Version: 1.0

Status: Final

Priority: Critical

---

# Purpose

This document defines the visual quality standard for EcoLoop.

AI developers tend to imitate common patterns.

This document explicitly defines:

* what to emulate
* what to avoid
* visual quality expectations
* design taste standards

This document is not optional.

---

# Hero References

Desired Feeling:

```txt id="w3k9nx"
I am looking into a living system.
```

---

Not:

```txt id="f8v2tp"
I am looking at a website.
```

---

Hero Must Include

At least:

* depth
* motion
* atmosphere

---

Hero Must Not Include

* stock images
* earth renders
* environmental illustrations
* generic SVGs

# Emotional Test

Every screen should pass:

---

Question 1

Does this look like another climate app?

If yes:

Fail.

---

Question 2

Would someone screenshot this?

If no:

Improve.

---

Question 3

Could this belong to:

```txt id="m7q4tw"
Apple

Linear

Arc
```

If not:

Improve.

---

Question 4

Does the interface reveal invisible systems?

If not:

Improve.

---

# Final Visual Benchmark

The goal is not:

```txt id="g8k2pr"
beautiful climate software
```

The goal is:

```txt id="r5n9vw"
beautiful software

that happens to be about climate
```

That distinction matters.

# Definition of Done

Visual quality is acceptable only when:

* distinctive
* memorable
* premium
* accessible
* responsive
* not obviously generated by AI
